Доски и колонки
===============

Выбор доски
-----------

После входа пользователь попадает на доску. Текущая доска хранится в сессии (и может совпадать с «доской по умолчанию» из профиля). В шапке приложения доступен переключатель досок: список досок, к которым у пользователя есть доступ, и выбор активной.

Вид доски (Kanban)
------------------

Доска отображается как набор колонок в одну линию. В каждой колонке — заголовок и список карточек. Карточки можно перетаскивать между колонками и внутри колонки (смена порядка); колонки при наличии прав можно переставлять (drag-and-drop).

*Скриншот: ``boards-kanban-view.png`` — несколько колонок с карточками, разный цвет границ по важности.*

Права на доску
--------------

Доступ к доске определяется членством (BoardMembership). Назначать пользователей на доски может администратор (в админ-кабинете, раздел «Пользователи» / доступ к доскам). Обычный пользователь видит только те доски, на которые его добавили.

Поиск по доске
--------------

В шапке доступен поиск. Результаты поиска отображаются в виде списка карточек (по текущей доске), по ним можно перейти к открытию карточки.

Управление колонками (только для администратора)
------------------------------------------------

Администратор может создавать, переименовывать, менять порядок и удалять колонки. Редактирование и перестановка колонок доступны в админ-кабинете (раздел «Доски» / выбранная доска) или в контексте доски, в зависимости от реализации UI. Архивирование колонки создаёт архив (см. :doc:`admin`).

Итог по API: выбор доски (POST /api/boards/select), список досок (GET /api/boards), контекст доски и пользователи доски (GET /api/board, GET /api/users); колонки и архивы колонок — в :doc:`api`.
