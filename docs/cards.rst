Карточки
========

Карточка — основная единица работы: задача с описанием, деталями, исполнителем, сроком, важностью и флагом «на паузе». К карточке можно добавлять комментарии, вложения и участников; участникам при включённых настройках могут отправляться уведомления по email.

Список карточек на доске
-----------------------

В колонках отображаются карточки в виде плиток (карточек). На плитке обычно видны: краткое описание, метки важности (низкая/средняя/высокая), срок, исполнитель, индикатор паузы. Цвет границы или бейджа может отражать важность.

Перетаскивание (Drag and Drop)
------------------------------

Карточки можно перетаскивать между колонками и менять их порядок внутри колонки. Перетаскивание реализовано на фронтенде (@dnd-kit) с отправкой нового положения на backend (например, POST /api/cards/:id/move с columnId и position).

Открытие и редактирование карточки
----------------------------------

По клику на карточку открывается модальное окно с полной информацией:

* **Описание и детали** — текстовые поля.
* **Исполнитель (assignee)** — выбор пользователя из участников доски.
* **Срок (dueDate)** — дата.
* **Важность** — низкая / средняя / высокая.
* **На паузе** — флаг.
* **Участники карточки** — список пользователей, подписанных на карточку (добавление/удаление через API).

Комментарии

В модальном окне карточки есть блок комментариев: список комментариев с автором и датой, форма добавления и при необходимости редактирование/удаление своих комментариев.

Вложения
--------

К карточке можно прикреплять файлы. Отображаются имя файла и возможность скачать или удалить вложение (в зависимости от прав). Загрузка — через форму в модальном окне карточки.

Создание и удаление карточек
----------------------------

* **Создание:** в колонке обычно есть кнопка «Добавить карточку» или аналогичная; вводится описание (и при необходимости остальные поля). Создание через API: POST /api/cards с boardId, columnId, description и др.
* **Удаление:** кнопка удаления в модальном окне карточки (DELETE /api/cards/:id).
* **Архивирование:** карточку можно отправить в архив (POST /api/cards/:id/archive); просмотр и восстановление архивов — в админ-кабинете (:doc:`admin`).

Важность и пауза
----------------

* **Важность (importance):** LOW / MEDIUM / HIGH; влияет на отображение (цвет, бейдж) и может использоваться для фильтрации.
* **Пауза (paused):** карточка может быть помечена как приостановленная; в интерфейсе это может отображаться иконкой или стилем.

Итог по API карточек: CRUD карточек, move, archive, participants, comments, attachments — см. :doc:`api`.
