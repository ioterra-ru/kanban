Развёртывание
=============

Требования
----------

* Docker и Docker Compose.
* (Опционально) домен и сертификаты для HTTPS.

Сборка и запуск одним приложением
---------------------------------

В корне проекта предусмотрены скрипты **run_one_app_dev.sh** (локально, compose base+dev) и **run_one_app_prod.sh** (продакшен, compose base+prod), которые поднимают приложение (frontend, backend, PostgreSQL и т.д.) через Docker Compose.

Конфигурация окружения
----------------------

Переменные окружения задаются в файлах в каталоге **docker/compose/**:

* **.cont_one_app.env** — основные переменные (URL приложения, порты и т.п.).
* **.cont_one_app.secrets.env** — секреты (пароли БД, секреты сессий и т.д.).

Перед первым запуском скопируйте примеры (если есть) и заполните значения.

Основные сервисы (docker-compose)
----------------------------------

* **PostgreSQL** — база данных для приложения и сессий.
* **Backend** — Node.js/Express API; подключается к БД по ``DATABASE_URL``.
* **Frontend** — собранное статическое приложение (например, nginx или сервис на Vite в dev).

Миграции БД выполняются при старте backend (Prisma migrate).

Резервное копирование и восстановление
---------------------------------------

В каталоге **scripts/** находятся скрипты:

* **backup-db.sh** — создание дампа базы данных.
* **restore-db.sh** — восстановление из дампа.
* **admin-reset-user-password.sh** — сброс пароля пользователя по email с сервера (без почты и UI-админа).

Используйте их согласно инструкциям в скриптах и в README проекта.

Логи
----

Для просмотра логов backend: ``docker compose`` с теми же ``--env-file``, что при запуске, например ``... logs -f backend``.

Документация по сборке документации
-----------------------------------

Сборка Sphinx-документации описана в :ref:`docs-build` (раздел «Сборка документации» в конце этого файла).

.. _docs-build:

Сборка документации
-------------------

#. Установите зависимости: ``pip install -r docs/requirements.txt`` (из корня проекта или из ``docs/`` в зависимости от расположения файла требований).
#. Перейдите в каталог ``docs/`` и выполните: ``make html`` (или ``sphinx-build -b html . _build/html``).
#. Откройте ``docs/_build/html/index.html`` в браузере.

Картинки для документации помещайте в ``docs/images/`` и подключайте в RST как ``.. image:: images/имя_файла.png``.
